# 멤버 함수
**멤버 함수**: 구조체에 포함된 함수  
> 구조체에 포함된 변수는 멤버 변수라고 부른다.    
> 다른 언어에서는 멤버 변수를 필드(Field), 멤버 함수를 메소드(Method)라고 부르기도 하는데 같은 뜻이다.  
> 구조체에 원형만 적고 본체는 바깥에 적을 수 있다.  
> 본체를 바깥으로 적을 경우 그냥 함수처럼 작성하되 멤버함수명 앞에 "소속구조체::" 를 붙인다.  
> 본체를 구조체 내부에 적을 경우 인라인 속성을 가져서 실제 함수가 호출되는게 아닌 멤버 함수를 호출하는 코드가 함수 본체 코드로 대체된다.
> 인라인 함수는 호출 부담이 없어서 빠르지만 여러 번 호출하면 파일의 크기가 증가된다.  
> 멤버 함수를 외부에 만들면서 인라인으로 만들고 싶으면 inline을 함수 원형이나 정의부의 리턴타입 앞에 쓰면 된다.  
> 멤버 함수를 클래스 내부에 정의하면 무조건 인라인 속성을 가진다.

# 액세스 지정
**private**: 이 속성을 가지는 멤버는 외부에서 액세스할 수 없으며 구조체의 멤버 함수만 액세스할 수 있다.   
**public**: 이 속성을 가지는 멤버는 외부로 공개되어 누구나 읽고 쓸 수 있고 함수의 경우는 호출할 수 있다.  
**protected**: private와 마찬가지로 외부에서는 액세스할 수 없으나 단, 상속된 파생 클래스는 이 멤버를 액세스할 수 있다.  
> 구조체가 자신의 속성이나 동작을 외부로 공개하는 수단이 되며 퍼블릭 멤버를 소위 인터페이스라고 한다.  
> 프라이비트 멤버는 파생 클래스에서조차도 참조할 수 없으며 오로지 자신만이 이 멤버를 참조할 수 있다는 점이 protected와 다르다.  
> 액세스 지정자는 구조체 선언문 내에서만 사용되는데 다른 액세스 지정자가 나올 때까지 계속 이 속성이 적용된다.  
> 액세스 지정자의 순서에 대한 제약은 없으며 필요에 따라 여러 번 중복될 수도 있다.  
> 액세스 지정자를 사용하여 특정 멤버를 숨길 수 있는 OOP의 기능을 정보 은폐라고 한다.

<pre>
#include <iostream>

struct Babo
{
private:
    int a;
    double b;
    char ch;
    void Initialize();

public:
    int x;
    int y;
    void func(int i);

protected:
    float k;
};

void main()
{
    Babo Kim;

    Kim.a = 1;             //에러
    Kim.x = 10;            //대입 가능
    Kim.func(3);           // 호출 가능
    Kim.Initialize();      // 에러
}
</pre>
